# Intent-First Visualization

프로젝트 구조·흐름·관계를 설명할 때, 표현 방식을 먼저 결정한 후 렌더링한다.
텍스트가 더 명확한 경우 시각화 도구를 쓰지 않는다.

---

## 핵심 원칙

**의도 → 형식 선택 → 렌더링 품질 확보** 순서로 진행한다.
도구가 있다고 무조건 쓰지 않는다. 형식은 사용자의 질문 구조가 결정한다.

---

## 1단계: 의도 파악 → 형식 선택

| 사용자 질문 유형 | 알고 싶은 것 | 선택 형식 | 이유 |
|--------------|-----------|---------|------|
| "구조가 어떻게 돼?" | 파일/폴더 계층 | **텍스트 트리** | 계층이 선형 텍스트로 충분히 전달됨 |
| "어떻게 동작해?" (단순) | 순서가 있는 단계 | **번호 목록** | 시각화 불필요 |
| "데이터가 어떻게 흐르지?" | 런타임 데이터 플로우 | **플로우 다이어그램** | 단계와 방향이 시각화로 명확해짐 |
| "모듈들이 어떻게 연결돼?" | 컴포넌트 간 의존성 | **구조 다이어그램** | 관계와 경계가 공간적으로 표현될 때 유리 |
| "테이블들이 어떻게 연결돼?" | DB 스키마 | **ERD** | 속성·관계·카디널리티가 많아 텍스트로 표현 불가 |
| "전체 시스템이 어떻게 생겼어?" | 서비스 간 아키텍처 | **아키텍처 다이어그램** | 경계·역할·통신 방향이 한눈에 보여야 함 |

---

## 2단계: 형식별 렌더링 품질 기준

### 텍스트 트리
- **깊이**: 3단계 이내. 더 깊으면 핵심 경로만 표시하고 생략 표기 (`...`)
- **주석**: 역할이 불명확한 파일은 옆에 `# 역할` 표기
- **그룹핑**: 연관 파일끼리 인접하게 배치

```
src/
├── main.py              # FastAPI 진입점
├── pipeline/
│   ├── runner.py        # 파이프라인 오케스트레이터
│   └── processor.py     # AI 호출
└── viewer/
    └── builder.py       # HTML 빌드
```

### 번호 목록 (단계 설명)
- **한 줄 = 한 동작**: 동작 주체 + 행위 + 결과
- **5단계 초과 시**: 단계를 그룹으로 묶어 소제목 추가
- **화살표 기호 금지**: `→` 대신 번호로 순서를 표현

### 플로우 다이어그램 (SVG)
- **노드 수**: 최대 5개. 초과 시 여러 다이어그램으로 분할
- **방향**: 단방향 고정 (위→아래 또는 왼→오른)
- **노드 레이블**: 역할 한 줄 + 선택적으로 부제목 한 줄
- **화살표 레이블**: 상태 변화가 있을 때만 표기. 명백한 흐름엔 생략

### 구조 다이어그램 (SVG)
- **노드 수**: 한 다이어그램에 최대 7개
- **색상**: 역할별로 구분, 최대 3가지 ramp
- **화살표**: 의존 방향만 표시. 양방향 화살표 금지
- **컨테이너**: 논리적 경계(서비스, 레이어)는 dashed rect로 표현

### ERD (mermaid)
- **표시 범위**: 핵심 관계만. 모든 FK를 표시하지 않음
- **테이블명**: 실제 스키마 이름 사용
- **분할**: 10개 이상 테이블이면 도메인 단위로 분할 후 각각 렌더링

---

## 3단계: 렌더링 전 자문

1. 텍스트로도 충분히 전달되는가? → YES면 텍스트 사용
2. 선택한 형식이 질문 유형과 일치하는가? → 불일치면 형식 재선택
3. 노드/단계 수가 기준을 초과하는가? → 초과면 분할
4. 렌더링 후 "이게 사용자 의도를 명확히 전달하는가?" → NO면 재작성

## 예시

### "파이프라인 구조가 어떻게 돼?"
- 의도: 파일/모듈 계층
- 형식: 텍스트 트리 (시각화 불필요)
- 품질 기준: 3단계 이내, 핵심 파일에 역할 주석

### "노트가 위키가 되기까지 어떻게 흐르지?"
- 의도: 런타임 데이터 플로우
- 형식: 플로우 다이어그램 (SVG)
- 품질 기준: 노드 5개 이내, 위→아래 방향, 각 노드에 역할 명시

### "Drive 인증이 어떻게 결정돼?"
- 의도: 조건 분기가 있는 로직
- 형식: 번호 목록 또는 플로우 (분기가 2개 이하면 목록으로 충분)
- 품질 기준: 조건과 결과를 한 줄씩 명확히
