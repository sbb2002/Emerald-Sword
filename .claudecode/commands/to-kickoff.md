# To-Kickoff

## Overview
* 사용법 :  `/to-kickoff [사용자의 아이디어]`
* 설명 : 사용자가 프로젝트 아이디어를 말하면 구현하기 전에 우선적으로 아래 ' Template'에 대해 서술하도록 합니다. 아이디어에서 아래 항목 중 답변 가능한 것은 다시 묻지말고, 만약 아이디어에서도 답변하지 못하는 것이 있으면 사용자에게 다시 되묻습니다. 모든 질문에 대해 답변이 완료되면 이를 바탕으로 'KICKOFF.md'를 작성합니다.

## Template
```
# [프로젝트명] 킥오프 문서

## 1. 핵심 가치
이걸 왜 만드는가? 한 문장으로.

## 2. MVP 범위
딱 이것만. 그 외는 명시적으로 제외.

## 3. UI 앵커
화면/인터페이스가 어떻게 생겼는지. 
(범위가 눈에 보이게 하는 장치)

## 4. 기술 선택 근거
익숙해서인가, 적합해서인가.

## 5. 가장 먼저 무너질 것
구조적 약점 2개.

## 6. Kill Condition
이 조건이 충족되면 미련 없이 중단한다.

## 7. 의사결정 로그
(개발 중 채워나갈 것 — 킥오프 시점에는 비워둠)
형식: [날짜] [결정 내용] / [이유] / [포기한 대안]

```
