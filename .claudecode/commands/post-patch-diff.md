# Post-patch Diff

패치 완료 후 선언된 파일 목록 대비 실제 변경 사항을 대조하는 스킬.

## 목적

여러 파일을 동시에 수정할 때 **일부 파일이 반영되지 않는 것**이 가장 흔한 실패다.
Post-patch Diff는 pre-flight에서 합의한 목록을 기준으로 실제 반영 여부를 검증한다.

---

## 실행 순서

### 1. 기준선 불러오기

pre-flight-check에서 선언한 파일 목록을 참조한다.
pre-flight를 건너뛴 경우 → 지금 작업한 파일 목록을 직접 재구성한다.

### 2. 환경별 검증

#### Claude.ai 아티팩트

파일시스템이 없으므로 **대화 내에서 선언적으로 검증**한다:

```
## Post-patch Diff

### 변경 계획 vs 실제 반영

| # | 파일/컴포넌트 | 계획 | 반영 | 비고 |
|---|---------------|------|------|------|
| 1 | Button.tsx | 클릭 핸들러 추가 | ✅ | - |
| 2 | button.css | hover 스타일 | ✅ | - |
| 3 | types/index.ts | ButtonProps 확장 | ❌ | 누락 — 지금 반영 |

### 예상 외 변경
- 없음

→ ❌ 1건 누락: types/index.ts 즉시 패치
```

---

#### Claude Code / 로컬

git이 초기화된 경우:

```bash
# 변경된 파일 목록 확인
git diff --name-only

# 특정 파일 변경 내용 확인
git diff src/components/Button.tsx

# 미추적 파일 확인 (신규 파일)
git status --short
```

git이 없는 경우:
```bash
# 수정 시간 기준으로 최근 변경 파일 확인
find . -newer /tmp/patch-start-marker -type f \
  ! -path "*/node_modules/*" \
  ! -path "*/.git/*"
```

타임스탬프 마커 생성 (작업 시작 전):
```bash
touch /tmp/patch-start-marker
```

---

### 3. 결과 분류 및 조치

| 상태 | 의미 | 조치 |
|------|------|------|
| ✅ 반영됨 | 계획대로 변경 완료 | 없음 |
| ❌ 누락 | 계획에 있었으나 미반영 | 즉시 패치 |
| ⚠️ 예상 외 변경 | 계획에 없던 파일 변경 | 의도적인지 확인 |
| ➖ 불필요 | 계획에 있었으나 실제론 불필요 | 기록만 |

---

## 결과 보고 형식

```
## Post-patch Diff 결과

반영: 3 / 계획: 4
누락: 1건 — src/types/index.ts
예상 외 변경: 없음

→ ❌ 패치 미완료: 누락 파일 처리 후 재검증
```

또는:

```
## Post-patch Diff 결과

반영: 4 / 계획: 4
누락: 없음
예상 외 변경: 없음

→ ✅ 패치 완료. syntax-gate 또는 실행으로 넘어가도 됩니다.
```

---

## 규칙

- ❌ 누락이 있으면 **즉시 해당 파일 패치** 후 해당 항목만 재검증
- ⚠️ 예상 외 변경은 **의도적인지 확인** 후 기록에 남긴다
- 완료 판정은 **모든 항목 ✅** 일 때만
- 완료 후 자연스럽게 `give-me-receipt` 또는 `get-issued` 로 연결 가능

---

## pre-flight 없이 사용할 때

pre-flight를 건너뛴 경우, 작업 시작 전에 다음을 먼저 재구성한다:

```
pre-flight 기록이 없습니다.
이번 작업에서 건드린 파일을 열거해주세요,
또는 제가 대화 내역에서 추정한 목록을 확인해주세요:

- [추정 파일 1]
- [추정 파일 2]

맞나요?
```
