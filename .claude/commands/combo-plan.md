# Combo Plan

아이디어 → 이슈 분해까지 기획 전 단계를 하나의 명령으로 실행하는 오케스트레이터 스킬.

## 사용법

```
/combo-plan [아이디어 또는 설명]
```

아이디어 없이 `/combo-plan`만 쳐도 된다 — grill-me가 먼저 끌어낸다.

---

## 실행 흐름

```
① grill-me        — 아이디어 인터뷰 (자동 종료)
        ↓
② to-kickoff      — KICKOFF.md 생성
        ↓
③ to-prd          — PRD 작성
        ↓
④ to-issues       — 이슈 분해 (내부 확인 루프 포함)
        ↓
⑤ combo-run 실행 여부 질문
```

---

## 단계별 진행 규칙

### ① grill-me
- grill-me 스킬의 인터뷰 종료 조건이 충족될 때까지 질문을 이어간다.
- 종료 조건: 결정 트리의 모든 주요 분기 해소 / 사용자가 종료 신호 전달
- 종료 후 "공유된 이해 요약"을 출력하고 ②로 자동 진행한다.

### ② to-kickoff
- grill-me 요약을 인풋으로 삼아 KICKOFF.md를 작성한다.
- 이미 답변된 항목은 다시 묻지 않는다.
- 완료 후 ③으로 자동 진행한다.

### ③ to-prd
- KICKOFF.md를 기반으로 PRD를 작성한다.
- 완료 후 ④로 자동 진행한다.

### ④ to-issues
- PRD를 기반으로 이슈를 분해한다.
- to-issues 내부의 확인 루프(초안 제시 → 사용자 피드백 → 수정)를 그대로 따른다.
- 사용자가 이슈 목록을 승인하면 publish 후 ⑤로 진행한다.

### ⑤ combo-run 연결
이슈 publish 완료 후 아래 메시지를 출력하고 대기한다:

```
## Combo Plan 완료

총 N개의 이슈가 생성됐습니다.

이제 실행 단계로 넘어갈 수 있어요:
- `/combo-run`       — 전체 이슈를 순서대로 실행
- `/combo-run N`     — 이슈 N번 단일 실행
- `/combo-run --list` — 이슈 목록 보고 선택

바로 시작할까요?
```

---

## 중단 및 재개

- 어느 단계에서든 사용자가 중단하면 현재 단계의 결과물을 보존한다.
- 재개 시 중단된 단계부터 이어서 진행한다.
- 예: to-prd 작성 중 중단 → 재개 시 to-prd 결과 확인 후 to-issues부터 시작.
